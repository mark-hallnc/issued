# Invite Links

Workspace invite emails point to:

```text
https://issuedinventory.com/invite?token=<invite_token>
```

The token is generated by the `invite-workspace-member` Edge Function and is
accepted by the authenticated RPC `public.accept_workspace_invite_by_token`.
The Flutter app stores the token locally, asks the user to sign in when needed,
and accepts the invite after the signed-in email matches the invited email.

## Android App Links

The Android app is configured to open:

- `https://issuedinventory.com/invite?token=<invite_token>`
- `issued://invite?token=<invite_token>`

To make HTTPS App Links open directly into the installed app, host an Android
Digital Asset Links file at:

```text
https://issuedinventory.com/.well-known/assetlinks.json
```

That file must include this app package and the release signing certificate
fingerprint:

```json
[
  {
    "relation": ["delegate_permission/common.handle_all_urls"],
    "target": {
      "namespace": "android_app",
      "package_name": "com.issuedinventory.issued_app",
      "sha256_cert_fingerprints": ["<release-signing-sha256>"]
    }
  }
]
```

## Web Fallback

Host a simple page at:

```text
https://issuedinventory.com/invite
```

Suggested copy:

```text
You've been invited to Issued.

Open the Issued app and sign in with the email that received this invite.
If Issued is installed, tap Open Issued.
```

Until Android App Links are fully verified, users can still open Issued
manually and sign in with the invited email address. The app will accept any
pending invite token that was previously opened on the device.

## Edge Function Configuration

The invite Edge Function reads:

- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`
- `RESEND_API_KEY`
- `INVITE_EMAIL_FROM`, optional, defaults to
  `Issued <noreply@issuedinventory.com>`
- `PUBLIC_INVITE_BASE_URL`, optional, defaults to
  `https://issuedinventory.com/invite`

Do not use Supabase Auth callback URLs for workspace invite links, and do not
put the service role key in Flutter.
